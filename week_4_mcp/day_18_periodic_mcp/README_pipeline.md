# Task 19 — MCP Tool Composition Pipeline

Три MCP-инструмента, выстроенные в пайплайн:
**search → summarize → save**.

Агент сам вызывает цепочку — пользователь пишет один запрос на естественном языке.

---

## Архитектура пайплайна

```
Пользователь
    │  "Собери сводку новостей HN про AI и сохрани"
    ▼
agent_with_mcp.py  (DeepSeek, function calling)
    │
    │  Tool call 1: search_hn(limit=20, keyword="AI")
    ▼
hn_mcp_server.py ──► HN Firebase API
    │  возвращает JSON-список историй
    │
    │  Tool call 2: summarize_stories(stories_json=..., top_n=10)
    ▼
hn_mcp_server.py
    │  возвращает отформатированный текст
    │
    │  Tool call 3: save_to_file(content=..., filename="ai_digest.txt")
    ▼
hn_mcp_server.py ──► data/digests/ai_digest.txt
    │
    ▼
agent_with_mcp.py — финальный ответ пользователю
```

**Ключевое:** каждый инструмент делает одно. Выход первого — вход второго.
Агент сам строит цепочку, исходя из запроса пользователя.

---

## Три пайплайновых инструмента

### 1. `search_hn` — получить данные

Обращается к HN Firebase API напрямую. **Не сохраняет** в БД — только возвращает JSON.

| Параметр | Тип | Умолчание | Описание |
|---|---|---|---|
| `limit` | int | 20 | Сколько историй получить (макс. 100) |
| `keyword` | str | `""` | Фильтр по слову в заголовке |
| `min_score` | int | 0 | Минимальный рейтинг |

Возвращает: JSON-строку со списком историй.

```json
[
  {
    "hn_id": 12345,
    "title": "Ask HN: What's new in AI?",
    "url": "https://...",
    "score": 342,
    "comments": 87,
    "author": "someuser",
    "hn_url": "https://news.ycombinator.com/item?id=12345"
  },
  ...
]
```

### 2. `summarize_stories` — обработать данные

Принимает JSON из `search_hn`, сортирует по рейтингу, форматирует в читаемый текст.

| Параметр | Тип | Умолчание | Описание |
|---|---|---|---|
| `stories_json` | str | — | JSON из `search_hn` |
| `top_n` | int | 10 | Сколько лучших историй включить |
| `style` | str | `"digest"` | Формат: `digest` / `brief` / `full` |

Стили вывода:
- **`digest`** — рейтинг + комментарии + заголовок + ссылка (по умолчанию)
- **`brief`** — только нумерованный список заголовков
- **`full`** — все поля включая автора и HN-ссылку

Возвращает: текстовую строку (готова к сохранению или выводу).

```
HackerNews Top-10 | 2026-06-27 14:30
==================================================

1. ⬆342 💬87  Ask HN: What's new in AI?
   https://news.ycombinator.com/item?id=12345
2. ⬆289 💬54  LLM inference at 10x speed
   https://example.com/llm-speed
...
```

### 3. `save_to_file` — сохранить результат

Сохраняет текст в файл в папку `data/digests/`. Имя файла — опционально.

| Параметр | Тип | Умолчание | Описание |
|---|---|---|---|
| `content` | str | — | Текст для сохранения |
| `filename` | str | `""` | Имя файла (auto: `digest_YYYYMMDD_HHMMSS.txt`) |

Возвращает: путь к файлу и размер.

```
Сохранено: /path/to/data/digests/ai_digest.txt (1243 символов, 18 строк)
```

## Примеры запросов агенту

Агент сам решает, какие инструменты вызвать и в каком порядке.

### Полный пайплайн (все 3 шага) — надёжный промпт:

```
Прямо сейчас получи свежие данные с HackerNews API (не из базы),
составь красивую сводку топ-10 историй по рейтингу
и сохрани результат в текстовый файл. Покажи путь к файлу.
```
→ `search_hn(limit=20)` → `summarize_stories(top_n=10)` → `save_to_file()`

**Почему этот промпт работает надёжно:**
- "прямо сейчас" + "из API" + "не из базы" → агент выбирает `search_hn`, а не `collect_now`
- нет фильтра по теме → список всегда непустой, нет риска пустого ответа
- "в текстовый файл" → агент выбирает `save_to_file`, а не просто показывает текст

> **Осторожно с фильтрами по теме.** Запрос "найди новости про Python" может вернуть
> пустой список, если Python не в топе прямо сейчас. Тогда агент может запутаться
> и попытаться вызвать несуществующие инструменты. Используй широкие темы или
> убирай фильтр совсем.

### Только получить данные (шаг 1):
```
Какие сейчас топ-новости на HackerNews? Получи их из API прямо сейчас.
```
→ `search_hn(limit=20)`

### Получить + показать без сохранения (шаги 1-2):
```
Прямо сейчас получи топ-новости HackerNews из API и покажи краткую сводку 5 лучших.
```
→ `search_hn(limit=20)` → `summarize_stories(top_n=5, style="brief")`

## Как работает автоматическая цепочка

1. Пользователь пишет запрос на естественном языке
2. DeepSeek получает список всех инструментов с описаниями
3. LLM решает вызвать `search_hn` первым (нужны данные)
4. Получает JSON → передаёт как аргумент в `summarize_stories`
5. Получает текст → передаёт как аргумент в `save_to_file`
6. Агент формирует финальный ответ пользователю

Каждый шаг — отдельный `tool_call` в истории диалога DeepSeek.
Данные между шагами передаются через аргументы: выход инструмента N → вход инструмента N+1.

```
messages = [
  {role: "user",      content: "Собери сводку..."},
  {role: "assistant", tool_calls: [search_hn(...)]},
  {role: "tool",      content: "[{hn_id:..., title:...}]"},   ← JSON историй
  {role: "assistant", tool_calls: [summarize_stories(stories_json=<выше>)]},
  {role: "tool",      content: "HackerNews Top-10 | ..."},    ← текст сводки
  {role: "assistant", tool_calls: [save_to_file(content=<выше>)]},
  {role: "tool",      content: "Сохранено: data/digests/..."},
  {role: "assistant", content: "Готово! Сохранил сводку 10 новостей..."}
]
```

## Отличие от оригинальных инструментов

| | Оригинальные | Пайплайновые |
|---|---|---|
| `collect_now` | fetch + save в БД (2 действия) | — |
| `search_hn` | — | только fetch, возвращает JSON |
| `get_digest` | читает из БД + форматирует | — |
| `summarize_stories` | — | принимает JSON, форматирует |
| Сохранение | только SQLite (внутри `collect_now`) | только файл (`save_to_file`) |
| Зависимость | требует предварительного `collect_now` | независима, данные «на лету» |

Оба набора инструментов доступны одновременно. Агент выбирает нужные под задачу.
