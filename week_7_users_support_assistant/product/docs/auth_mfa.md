# Двухфакторная аутентификация (2FA / TOTP)

TaskPilot поддерживает второй фактор по стандарту TOTP: Google Authenticator, Authy,
Microsoft Authenticator, 1Password и любые совместимые приложения. 2FA доступна на **всех
тарифах**, включая Free.

## Включение

1. **Настройки → Безопасность → Двухфакторная аутентификация → Включить**.
2. Отсканируйте QR-код приложением-аутентификатором.
3. Введите текущий шестизначный код для подтверждения.
4. Сохраните **10 резервных кодов** — каждый одноразовый, показываются один раз.

Администратор может потребовать 2FA для всех участников:
**Настройки → Безопасность → Требовать 2FA**. Участники без второго фактора будут обязаны
настроить его при следующем входе.

## Ошибка MFA-401 — «Код подтверждения не принят»

Признаки: пароль принят, но шестизначный код отклоняется, хотя вводится верно и без опечаток;
часто ошибка появляется внезапно у пользователя, который раньше входил без проблем.

Причина в подавляющем большинстве случаев — **рассинхронизация времени на устройстве**. Коды
TOTP вычисляются из текущего времени, окно допуска — ±30 секунд. Если часы телефона ушли даже
на минуту, все коды будут отклоняться.

Что делать:

1. **Синхронизируйте часы устройства.** Включите автоматическое определение времени:
   - Android: **Настройки → Система → Дата и время → Автоматически**;
   - iOS: **Настройки → Основные → Дата и время → Автоматически**;
   - Google Authenticator: **Меню → Настройки → Коррекция времени для кодов → Синхронизировать**.
2. Дождитесь появления нового кода и введите его целиком, пока таймер не истёк.
3. Не помогло — войдите по **резервному коду** (ссылка «Использовать резервный код» под полем
   ввода). Каждый резервный код срабатывает один раз.
4. Резервные коды потеряны — обратитесь к администратору рабочего пространства: он может сбросить
   второй фактор в **Настройки → Команда → участник → Сбросить 2FA**. После сброса потребуется
   настроить аутентификатор заново.

## Ошибка MFA-429 — «Слишком много попыток ввода кода»

После 10 неверных кодов подряд ввод второго фактора блокируется на 15 минут. Дождитесь
окончания блокировки или войдите по резервному коду.

## Отключение 2FA

**Настройки → Безопасность → Двухфакторная аутентификация → Отключить**, с подтверждением
паролем. Если администратор включил требование 2FA для всех, отключить второй фактор нельзя.

## 2FA и SSO

При входе через корпоративный SSO второй фактор запрашивает провайдер идентификации, а не
TaskPilot: настройки 2FA в TaskPilot в этом случае не применяются.
