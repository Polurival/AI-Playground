# REST API: ключи и лимиты

Публичный REST API TaskPilot: `https://api.taskpilot.example/v1`.

## Выпуск API-ключа

1. **Настройки → API-ключи → Создать ключ**.
2. Задайте название и права: `read`, `write`, `admin`.
3. Скопируйте ключ сразу — он показывается один раз, в открытом виде больше не доступен.
4. Ключ передаётся в заголовке:

```
Authorization: Bearer tp_live_XXXXXXXXXXXXXXXX
```

Ключи привязаны к рабочему пространству, а не к пользователю. Срок жизни ключа — 12 месяцев,
за 14 дней до истечения владельцу приходит письмо.

## Ошибка API-401 — «API-ключ не принят»

Возможные причины:

- ключ отозван (**Настройки → API-ключи → Отозвать**);
- ключ истёк (12 месяцев с момента выпуска);
- ключ скопирован не полностью или с пробелом на конце;
- используется ключ другого рабочего пространства;
- отсутствует префикс `Bearer ` в заголовке `Authorization`.

Решение: выпустите новый ключ и замените его в интеграции. Отозванный ключ восстановить нельзя.

## Ошибка API-403 — «Недостаточно прав»

Ключ действителен, но у него нет нужного права: например, ключ с правом `read` вызывает
`POST /v1/tasks`. Права ключа изменить нельзя — выпустите новый ключ с нужными правами.

## Ошибка API-429 — лимит запросов

Лимит запросов зависит от тарифа рабочего пространства и считается **на рабочее пространство**
(не на ключ), окно — одна минута, счётчик скользящий:

| Тариф | Лимит |
|---|---|
| Free | 60 запросов в минуту |
| Pro | 600 запросов в минуту |
| Business | 3000 запросов в минуту |

При превышении API возвращает `429 Too Many Requests` (код `API-429`) и заголовки:

- `Retry-After` — через сколько секунд можно повторить запрос;
- `X-RateLimit-Limit` — лимит текущего тарифа;
- `X-RateLimit-Remaining` — сколько запросов осталось в текущем окне.

Что делать:

1. Реализуйте повтор с экспоненциальной задержкой, ориентируясь на `Retry-After`.
2. Уберите опрос по кругу: вместо периодических `GET /v1/tasks` подпишитесь на вебхуки
   (`docs/webhooks.md`) — события приходят сами и лимит не тратят.
3. Запрашивайте страницами по 100 элементов (`?limit=100`) вместо множества мелких запросов.
4. Постоянно упираетесь в лимит — поднимите тариф: Pro даёт 600 запросов в минуту,
   Business — 3000.

Отдельные лимиты, не зависящие от тарифа: `POST /v1/import` — 5 запросов в минуту,
`POST /v1/auth/token` — 20 запросов в минуту.

## Пагинация и версии

Ответы списков — `?limit=` (максимум 100) и `?cursor=`. Версия API фиксируется в пути (`/v1`),
ломающие изменения выходят только в новой версии.
