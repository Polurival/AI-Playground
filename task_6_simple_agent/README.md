# AI Агент с памятью и аналитикой токенов (DeepSeek API)

Консольный чат-бот на Python с DeepSeek API, отслеживающий использование токенов и контроль контекста.

## Требования

- Python 3.8+
- `openai` — OpenAI Python клиент (совместим с DeepSeek API)
- `tiktoken` — для подсчета токенов

## Установка

```bash
pip install -r requirements.txt
```

Или вручную:

```bash
pip install openai tiktoken
```

## Запуск

### Обычный чат с метриками токенов

```bash
export DEEPSEEK_API_KEY="your-api-key-here"
python main.py
```

Или в одну строку:

```bash
DEEPSEEK_API_KEY="your-api-key-here" python main.py
```

На Windows (PowerShell):

```powershell
$env:DEEPSEEK_API_KEY="your-api-key-here"
python main.py
```

### Демонстрация роста токенов и симуляции лимита

```bash
DEEPSEEK_API_KEY="your-api-key-here" python demo_tokens.py
```

### Команды в чате

- `/status` — показать использование контекста и процент заполнения лимита

- `Ctrl+C` — прерывание программы

## Структура кода

### `agent.py`

Класс `DeepSeekAgent` с функциями:

- **`__init__(system_prompt, history_file)`** — инициализация с опциональным системным промптом
- **`send_message(user_text)`** — отправка сообщения в API, возвращает `(ответ, метрики)`
- **`get_status()`** — получить статус использования контекста
- **`_count_tokens(messages)`** — подсчет токенов в сообщениях

Особенности:
- Автоматическое сохранение истории в JSON
- Подсчет токенов перед отправкой в API
- Симуляция лимита контекста (1000 токенов, настраивается)
- Получение реальных данных о токенах из DeepSeek API

### `main.py`

Интерактивный CLI:

- Запуск агента с восстановлением истории
- Красивый вывод метрик после каждого ответа
- Команда `/status` для мониторинга контекста
- Обработка ошибок переполнения контекста

### `demo_tokens.py`

Демонстрационный скрипт:

- Показ роста контекста при длинном диалоге
- Примеры метрик использования токенов
- Демонстрация срабатывания лимита контекста

## Метрики токенов

После каждого ответа выводятся метрики:

```
[Аналитика токенов]
  - Текущий запрос пользователя: X токенов
  - Всего в истории (контекст отправки): Y токенов
  - Ответ модели: Z токенов
  - Итого за этот шаг (Вход + Выход): N токенов
```

- **Текущий запрос** — токены текущего сообщения пользователя
- **Всего в истории** — размер контекста перед отправкой (включая все прошлые сообщения)
- **Ответ модели** — реальное количество токенов в ответе (из API)
- **Итого за шаг** — сумма входных и выходных токенов

## Контроль контекста

Агент симулирует лимит контекста (по умолчанию **1000 токенов**):

- Перед отправкой в API проверяется размер контекста
- Если контекст превышает лимит, сообщение **не добавляется** в историю
- Выбрасывается ошибка с информацией о текущем использовании
- История остается целой и готова к новому диалогу

Пример:

```
Вы: [длинное сообщение]
⚠ Ошибка: Достигнут лимит контекста (1245 из 1000 токенов). Диалог заблокирован.
```

## Конфигурация

### Изменить лимит контекста

```python
agent = DeepSeekAgent()
agent.max_context_tokens = 4000  # Новый лимит
```

### Использовать реальный лимит DeepSeek

```python
agent.max_context_tokens = 64000  # Или 128000 в зависимости от модели
```

## Примечания

- История диалога автоматически сохраняется в `chat_history.json`
- Каждый запрос отправляет полную историю на DeepSeek для сохранения контекста
- Подсчет токенов использует `tiktoken` с энкодером `cl100k_base` (совместим с DeepSeek)
- Реальные значения входных/выходных токенов получаются из API (100% точные)
- Для начала нового диалога удалите `chat_history.json`
