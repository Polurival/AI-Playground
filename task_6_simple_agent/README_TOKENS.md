# Аналитика токенов и контроль контекста в DeepSeek Agent

## Обзор

Обновленный агент теперь отслеживает использование токенов на каждом этапе диалога и симулирует ограничение контекста. Это позволяет:

- **Видеть рост расходов** — отслеживать, как растет стоимость запросов с каждым новым сообщением
- **Понимать поведение модели** — видеть реальное использование токенов из DeepSeek API
- **Тестировать лимиты** — симулировать сценарии переполнения контекста

## Что было добавлено

### 1. Подсчет токенов (`agent.py`)

#### Метод `_count_tokens(messages)`
```python
def _count_tokens(self, messages: list) -> int:
    """Подсчитывает токены в списке сообщений."""
    total_tokens = 0
    for msg in messages:
        content = msg.get("content", "")
        tokens = len(self.encoding.encode(content))
        total_tokens += tokens
    return total_tokens
```

- Использует `tiktoken` с энкодером `cl100k_base` (совместимо с DeepSeek)
- Подсчитывает токены до отправки запроса

#### Метод `get_status()`
```python
def get_status(self) -> dict:
    """Возвращает статус использования контекстного лимита."""
```

Возвращает:
- `used` — сколько токенов уже использовано
- `limit` — максимум (по умолчанию 1000 для симуляции)
- `percent` — процент заполнения
- `messages_count` — количество сообщений в истории

### 2. Симуляция лимита контекста

```python
self.max_context_tokens = 1000  # Можно менять для разных сценариев
```

**Логика проверки** в `send_message()`:
1. Подсчитываются токены всей истории (включая новое сообщение)
2. Если `history_tokens > max_context_tokens`:
   - Сообщение **не добавляется** в историю (откат)
   - Выбрасывается исключение `ValueError`
   - История остается нетронутой

**Пример ошибки:**
```
Ошибка: Достигнут лимит контекста (1245 из 1000 токенов). Диалог заблокирован.
```

### 3. Возвращаемое значение `send_message()`

Теперь метод возвращает **tuple** вместо строки:

```python
response, metrics = agent.send_message("Привет!")
```

Где `metrics` содержит:
- `current_query_tokens` — токены текущего сообщения
- `history_tokens_before_response` — размер контекста перед отправкой
- `prompt_tokens_used` — реальные токены входа (из API)
- `completion_tokens_used` — реальные токены выхода (из API)
- `total_this_step` — сумма входа и выхода

## Как это работает под капотом

### Источники данных для токенов

| Тип метрики | Источник | Точность |
|---|---|---|
| **Входные токены (до запроса)** | `tiktoken` с энкодером `cl100k_base` | Оценка ~95% |
| **Реальные входные токены** | `response.usage.prompt_tokens` из API | 100% точно |
| **Реальные выходные токены** | `response.usage.completion_tokens` из API | 100% точно |

### Логика проверки лимита

```
1. Пользователь вводит сообщение
   ↓
2. Подсчитываем токены текущего запроса
   ↓
3. Добавляем сообщение в историю (временно)
   ↓
4. Подсчитываем токены всей истории
   ↓
5. Сравниваем с лимитом:
   ├─ Если ≤ лимит → отправляем в API → сохраняем ответ → выводим метрики
   └─ Если > лимит → удаляем сообщение из истории → выбрасываем ошибку
```

## Интеграция с собственным кодом

```python
from agent import DeepSeekAgent

agent = DeepSeekAgent(system_prompt="Ты профессиональный переводчик")

try:
    response, metrics = agent.send_message("Переведи 'Hello' на русский")
    
    print(f"Ответ: {response}")
    print(f"Использовано входных токенов: {metrics['prompt_tokens_used']}")
    print(f"Использовано выходных токенов: {metrics['completion_tokens_used']}")
    
    # Проверяем статус
    status = agent.get_status()
    if status['percent'] > 90:
        print("Предупреждение: почти исчерпан контекст!")
        
except ValueError as e:
    print(f"Контекст переполнен: {e}")
```

## Решение проблем

### "ModuleNotFoundError: No module named 'tiktoken'"
```bash
pip install tiktoken
```

### "Ошибка: Достигнут лимит контекста" часто срабатывает
- Увеличьте `max_context_tokens` на агенте
- Или запустите новый диалог (удалите `chat_history.json`)

### Числа в метриках не совпадают
- `current_query_tokens` — оценка по `tiktoken` (может отличаться на несколько токенов)
- `prompt_tokens_used` — реальное число из API (точное)
- Разница нормальна и объясняется деталями кодирования
